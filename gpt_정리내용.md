# ReadMate 기획 정리

## 1. 서비스 개요

**ReadMate**는 시각장애인을 위한 지능형 학습 파트너입니다.  
종이책 이미지를 OCR로 읽고, LLM이 내용을 이해·요약·질문응답·퀴즈화한 뒤, TTS로 다시 들려주는 **AI 학습 보조 솔루션**입니다.

단순히 책을 읽어주는 것을 넘어,  
**읽기 → 이해 → 복습 → 질의응답**까지 연결하는 것이 핵심입니다.

## 3. 문제 정의

기존 OCR 앱이나 낭독 앱은 주로 아래 수준에 머무르는 경우가 많습니다.

- 글자를 읽어준다
- 긴 문장을 그대로 낭독한다
- 사용자가 내용을 이해했는지 확인하지 못한다
- 문맥 기반 질문응답이 어렵다

따라서 시각장애 학생이나 학습자에게는 단순한 텍스트 접근만으로는 부족하며,  
**구조 파악, 핵심 요약, 추가 설명, 복습 지원**까지 함께 제공되는 학습 보조 도구가 필요합니다.

---

## 4. 핵심 차별화 포인트

### 4.1 구조적 브리핑
본문을 읽기 전에 먼저 페이지 구조와 핵심 주제를 설명합니다.

예시:
> “이 페이지는 총 3개의 단락으로 구성되어 있으며, 핵심 주제는 인공지능의 역사입니다.”

이를 통해 사용자는 본문 전체를 듣기 전에 **큰 흐름**을 먼저 이해할 수 있습니다.

### 4.2 대화형 학습
읽다가 궁금한 개념을 바로 질문할 수 있습니다.

예시:
> “방금 읽은 부분에서 퍼셉트론이 정확히 뭐야?”

책 내용 기반으로 답변하여, 일반 사전식 설명이 아닌 **문맥에 맞는 설명**을 제공합니다.

### 4.3 능동적 복습
단원이나 페이지가 끝난 뒤 AI가 자동으로 퀴즈를 생성합니다.

- 객관식
- O/X
- 핵심 키워드 확인 문제

이를 통해 사용자는 **이해도 점검**을 할 수 있습니다.

### 4.4 쉬운 설명 및 정제
OCR 결과의 오인식 단어를 문맥에 맞게 보정하고, 어려운 문장을 더 쉬운 표현으로 풀어서 제공합니다.

즉,
- 텍스트를 단순 추출하는 데서 그치지 않고
- 학습에 적합한 형태로 재구성합니다

## 5. 전체 시스템 흐름

```text
이미지 입력 → 전처리 → OCR → 텍스트 정리 → LLM 분석 → TTS 출력 → 사용자 상호작용
```

### 5.1 이미지 입력
- 모바일 또는 웹 카메라로 책 페이지 촬영
- 파일 업로드 방식 지원

### 5.2 전처리
- OpenCV 기반 기울기 보정
- 그림자 제거
- 노이즈 완화
- 필요 시 본문/도표/이미지 영역 구분

### 5.3 OCR
- 책 페이지에서 텍스트 추출
- 줄/문단 단위로 병합 및 정리

### 5.4 LLM 처리
- 핵심 요약
- 키워드 추출
- 쉬운 설명
- 질의응답
- 퀴즈 생성
- OCR 오인식 교정

### 5.5 TTS 출력
- 요약본 또는 본문을 음성으로 변환
- 다시 듣기 기능 지원

### 5.6 사용자 인터페이스
- 큰 버튼 중심 설계
- 음성 우선 인터랙션
- 단순한 화면 흐름
- 접근성 친화적 UX

## 6. MVP 기술 스택

| 영역 | 적용 기술 | 역할 |
|---|---|---|
| Vision | PaddleOCR | 책 페이지 텍스트 추출 |
| LLM | Qwen2.5-7B-Instruct | 요약, 퀴즈 생성, 쉬운 설명, 문맥 교정 |
| Audio | MeloTTS | 한국어 중심 음성 출력 |
| UI | Streamlit / FastAPI | 데모 UI 및 API 서버 |
| 전처리 | OpenCV | 이미지 보정 및 OCR 성능 향상 |

### 확장 후보
| 영역 | 후보 기술 | 활용 방향 |
|---|---|---|
| OCR 고도화 | TrOCR | 인식 불안정 구간 재인식 |
| LLM 상향 | Qwen2.5-14B-Instruct | 요약 품질 및 문맥 유지 강화 |
| API 대안 | GPT-5.4-mini | 빠른 프로토타이핑 및 품질 검증 |
| TTS 고도화 | XTTS v2 / ElevenLabs | 더 자연스러운 음성 출력 |

## 7. 상세 기능 정리

### 7.1 텍스트 인식
- 책 페이지 이미지에서 글자 추출
- 한국어 문서 인식 중심

- 핵심 키워드 추출

- 어려운 개념을 쉽게 풀이

- 현재 읽은 본문을 기반으로 질문 가능

- 반복 청취 가능

## 8. 경쟁 서비스 대비 차별성

기존 낭독/OCR 앱은 보통 **“읽어주기”** 중심입니다.  
반면 ReadMate는 다음까지 지원합니다.

- 읽기 전 구조 브리핑
- 읽는 중 문맥 기반 질문응답
- 읽은 뒤 퀴즈 기반 복습
- OCR 결과 자동 정제 및 쉬운 설명

즉, 단순 접근성 도구가 아니라  
**학습 효율을 높이는 AI 학습 파트너**라는 점이 핵심 차별점입니다.

---

## 9. 접근성 및 UX 고려사항

시각장애인을 위한 서비스인 만큼, UI/UX에서도 접근성이 매우 중요합니다.

### 반영 포인트
- 큰 버튼
- 최소한의 화면 이동
- 음성 중심 안내
- 스크린 리더 친화 구조
- 현재 상태 음성 피드백
- 다시 듣기 / 다음 단계 이동 기능 명확화

이 부분은 단순 기능 구현보다도 서비스의 진정성과 완성도를 보여주는 핵심 요소입니다.

## 10. 추가 기술 제안

### 10.1 레이아웃 분석 고도화
책 페이지에서 아래 요소를 구분하면 품질이 크게 좋아질 수 있습니다.

- 제목 영역
- 본문 영역
- 이미지/도표 영역
- 표 영역

필요 시 YOLO 기반 커스텀 모델을 학습하여  
**본문 영역 / 제목 영역 분리 모델**을 직접 구축하는 것도 좋은 차별화 요소가 됩니다.

### 10.2 RAG 범위 제한
MVP 단계에서는 아래 중 하나로 범위를 제한하는 것이 좋습니다.

- 현재 페이지 기준
- 직전 N페이지 기준
- 현재 챕터 기준

이렇게 해야 구현 난이도와 답변 정확도를 함께 관리할 수 있습니다.

### 10.3 OCR 후처리 강화
책 촬영 이미지에서는 아래 문제가 자주 발생합니다.

- 기울어짐
- 그림자
- 페이지 곡면 왜곡
- 줄바꿈 파손
- 잡음

따라서 OCR 모델 자체보다도  
**전처리 + 문단 복원 + 후처리**가 최종 품질에 큰 영향을 줍니다.

## 11. TTS / OCR 조사 내용 정리

### 11.1 TTS 후보

#### MeloTTS
- 한국어 포함 다국어 지원
- 로컬 환경에 적합
- 빠른 응답 속도 기대 가능
- MVP용으로 적합

#### ElevenLabs
- 감정 표현이 풍부함
- 시연 임팩트가 큼
- API KEY 기반 데모 가능
- 상용 API 의존성이 있음

**정리:**  
- 로컬 MVP: `MeloTTS`
- 발표용 임팩트 시연: `ElevenLabs` 병행 가능

### 11.2 OCR 후보

#### PaddleOCR
- 문서형 이미지 처리에 적합
- MVP 시작점으로 가장 무난함

#### TrOCR
- 특정 줄/문단 재인식 보조 모델로 활용 가능

#### HDX-005
- 조사 후보로 언급되었으나, 현재 문서 기준에서는 참고 후보 정도로만 정리

## 12. 발표용 소개문

> **ReadMate는 시각장애인을 위한 지능형 학습 파트너입니다.**  
> 종이책을 촬영하면 OCR이 텍스트를 인식하고, LLM이 내용을 구조적으로 요약·설명·질의응답·퀴즈화하며, TTS가 이를 음성으로 전달합니다.  
> 단순히 책을 읽어주는 것을 넘어, 사용자가 내용을 이해하고 복습할 수 있도록 돕는 AI 학습 보조 솔루션입니다.

---

## 13. 발표용 핵심 요약 3줄

> 기존 OCR 앱이 “읽어주기”에 머문다면, ReadBridge는 “이해를 돕는 학습”까지 지원합니다.  
> 페이지 구조 브리핑, 문맥 기반 질의응답, 단원별 퀴즈 생성으로 학습 효율을 높입니다.  
> 시각장애 학생이 독서 접근성을 넘어 자기주도 학습까지 할 수 있도록 설계된 서비스입니다.

---

## 14. 최종 정리

ReadMate는 단순한 OCR 낭독 앱이 아니라,  
시각장애인을 위한 **AI 기반 학습 보조 솔루션**입니다.

핵심 경쟁력은 다음과 같습니다.

1. 단순 낭독이 아닌 구조적 이해 지원
2. 문맥 기반 대화형 학습
3. 퀴즈를 통한 능동적 복습
4. 접근성을 고려한 음성 중심 UX
5. 로컬 AI 기반 MVP 구현 가능성

즉, **“보이는 것을 읽는 서비스”가 아니라 “배움을 이어주는 서비스”**로 포지셔닝할 수 있습니다.
